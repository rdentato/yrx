YRX Patterns

   YRX does not entirely support the regular expression syntax. While YRX patterns offer extensions to RE matching, they do not support the * operator when applied to a pattern (it may be applied to a character or to a character class, though)..

   This limitation has a direct impact on the size of the DFA needed to recognize the patterns, see the documentation on the algorithm for further information.

Pattern List

   This is the summary list of terms to create yrx expression. You can read next section for a discussion of unusual terms
  pattern matches examples
. Any character "a.b" matches  "aob", "axb", "a1b", ....
[set] any of the specified characters "[akm]z" matches "az", "kz" or "mz".
[^set] any character but the specified ones "[^akm]z" matches "bz","cz", etc but not "az", "kz" and "mz"
[x-y] any character in the specified range "[a-b0-9]" matches "a", "0", "1", ... , "9","b"
\b the backspace character (ASCII 7)
\n the newline character (ASCII 10)
\r the carriage return character (ASCII 13)
\t the tab character (ASCII 9)
\xhh the char whose ASCII code is hh (in hexadecimal)  "\x41" matches "A"
\ddd the char whose ASCII code is ddd (in octal)  "\101" matches "A"
\a any letter (equivalent to [A-Za-z])
\l any lower case letter (equivalent to [a-z])
\u any upper case letter (equivalent to [A-Z])
\d any decimal digit (equivalent to [0-9])
\h any hexadecimal digit (equivalent to [0-9A-Fa-f])
\o any octal digit (equivalent to [0-7])
\s a space character (ASCII 32 or ASCII 9)
\w any letter (equivalent to [0-9A-Za-z_])
\A complement of \a (equivalent to [^A-Za-z])
\L complement of \l (equivalent to [^a-z])
\U complement of \u (equivalent to [^A-Z])
\D complement of \d (equivalent to [^0-9])
\H complement of \h (equivalent to [^0-9A-Fa-f])
\O complement of \o (equivalent to [^0-7])
\S complement of \s
\W complement of \w (equivalent to [^0-9A-Za-z_])
\Nmatcha  possibly signed number
\QMatches a single or double quoted string
\ematch any possibly escaped caractermatches "\q" as a single character
\Exset x as escape character 
p\:q matches p only if followed by q and pushes q back
\char match char literally
(pat) capture the text that matches pat
(pat)?capture the text that matches pat or the empty string
(p|q|...)matches p or q or ...
char the character
pat* matches 0 or more instance of pat
pat+ matches 1 or more instance of pat

p-qmatches the shortest sequence of p followed by q
